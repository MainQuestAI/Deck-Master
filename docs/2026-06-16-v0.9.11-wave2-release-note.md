# Deck Master v0.9.11 Wave 2 收口说明

发布日期：2026-06-16

## 变更摘要

v0.9.11 的 Wave 2 收口聚焦在两类交付：

- Benchmark 可观测性与 RC 判定边界收口。
- Deck Master 运行时文档路径收口，明确“Deck Master”为顶层编排入口。

## 主要更新

### 1) benchmark 报表与 RC 门槛

- `benchmark-report` 仍支持在早期链路生成，允许 run 仍处于非最终状态（如外部待处理项存在）。
- 新增 `benchmark-rc-report`，仅用于 v1.0 RC 评估：
  - 必须满足 `run_state` 的 `ready_for_benchmark`。
  - 写入文件为 `benchmark_rc_report.json` / `benchmark_rc_report.md`。
- fixture benchmark 运行在请求层面写入 `run_mode=fixture`，不会进入 v1.0 RC 判断通道。

### 2) 运行时运行模式一致性

- `benchmark-run` 在创建基准 run 时会根据 case 信息写入 `run_mode`：
  - fixture benchmark → `fixture`
  - 常规 benchmark → `benchmark`
- 该处理用于和 run_state 的生产/RC 读数保持一致。

### 3) Skill 使用指引

- `skills/deck-master/SKILL.md` 与运行说明路径收口到：
  - `start`
  - `doctor`
  - `setup-status`
  - `run-state`
  - `bind-workspace`
  - `orchestration-check`
- 强调真实 run 必须先通过 setup/workspace 绑定，再进入 planning / sourcing / quality / export。

## 使用约束（对外可见）

- 用户命名 Deck Master 时，Deck Master 为主控编排器；外部工具产物必须回写。
- fixture benchmark 仅用于演练和非 RC 场景。
- v1.0 RC 报表链路应优先调用 `benchmark-rc-report`，并以 `ready_for_benchmark` 为准。

## 风险与说明

- 现有 `benchmark-report` 命令仍保留兼容，适配补充分析场景。
- `benchmark-rc-report` 在尚未满足 RC 条件时会中止，便于避免不完整 run 误入版本门槛判断。

# Task F — Review Cockpit 2.0

## Goal
Upgrade the localhost UI from page preview to Run Review & Decision Cockpit.

## Scope
Implement:
- Deck Readiness Panel.
- Claim Coverage Matrix.
- Next 5 Actions.
- Page Decision Workbench.
- Review APIs.

## Delivery Phases
- F1 read-only APIs: review summary, claim coverage, next actions. Reuse current narrative, asset signals, and quality governance data.
- F2 page workbench actions: approve, reject, request evidence, convert to generate, lock source, add note.
- F3 external result visibility: show narrative advice, external review, and generation result after Packages C/D/E are stable.

Do not implement all phases in one PR. F1 is the first required slice.

## Allowed files
- `scripts/preview/server.py`
- `scripts/preview/manifest.py`
- `scripts/preview/static/app.js`
- `scripts/preview/static/index.html`
- `scripts/preview/static/style.css`
- `scripts/review/*.py`
- `tests/test_review_cockpit.py`

## Requirements
- Show deck readiness.
- Show claim coverage.
- Generate next actions from quality findings, evidence gaps, generation status, manual placeholders.
- Support review actions: approve, reject, request evidence, convert to generate, move to appendix, lock source, create override, rerun generation, add note.
- All actions write typed events.
- Review cannot bypass Quality Gate.
- External review and generation result visible.

## Tests
- Review summary API.
- Claim coverage API.
- Next actions API.
- Page review action API.
- Event logging.
- Quality blocking remains enforced.

## Done for F1
User can open the Cockpit and immediately see run readiness, claim coverage, and the next five highest-priority actions without changing any page state.

## Done
User can understand why a deck is not ready and act on the most important issues from the Cockpit.

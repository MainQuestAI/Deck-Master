# Task E — Build Tool Handoff / Handback Contract

## Goal
Strengthen the contract between Deck Master and PPT Deck Pro Max / PPT Master without embedding generation.

## Scope
Implement:
- `prepare-generation-handoff`
- `import-generation-result`
- `refresh-preview-from-generation`
- generation result validator

## Allowed files
- `scripts/generation/handback.py`
- `scripts/deck_master.py`
- `skills/deck-master/schemas/generation_result.schema.json`
- `skills/deck-master/playbooks/ppt-deck-pro-max-handoff.md`
- `tests/test_generation_handback.py`

## Requirements
- Deck Master does not generate slides.
- Completed result updates generation status and preview_manifest.
- Failed result visible in Review Cockpit.
- Partial result allowed but marked.
- Locked pages cannot be overwritten unless force.
- Bad result rejected safely.
- Write typed events.

## Tests
- Completed import.
- Failed import.
- Partial import.
- Locked page protection.
- Bad JSON rejection.
- Preview refresh.

## Done
PPT Deck Pro Max can produce a result JSON and Deck Master imports it into the run.

# Task I — Lightweight Metrics Hooks

## Goal
Prepare for v1.0 benchmark without building a heavy benchmark system now.

## Scope
Implement:
- `summarize-run-metrics`
- `run_metrics.json`

## Allowed files
- `scripts/metrics/run_metrics.py`
- `scripts/deck_master.py`
- `tests/test_run_metrics.py`

## Requirements
- Compute from events and artifacts.
- Support both legacy `append_event` records and canonical `append_typed_event` records.
- Prefer typed events when duplicate semantic actions exist.
- Degrade gracefully when events are missing.
- Include run timeline, counts, sourcing stats, review stats, quality stats.
- Does not block main runtime.
- No benchmark dashboard.

## Tests
- Complete events.
- Mixed legacy and typed events.
- Missing events fallback.
- Quality counts.
- Sourcing counts.
- Approved/rejected counts.

## Done
Deck Master can generate run-level metrics needed for future v1.0 10x benchmark.

# Task B — Agent Context Pack Contract

## Goal
Let external Agents parse/prepare real materials and hand structured context to Deck Master.

## Scope
Implement:
- `deck_context_pack.v1` schema.
- `import-context-pack`.
- `create-run-from-context-pack`.

## Existing Compatibility
- Keep the existing `connector import` command and `deck_connector_import.v1` tests working.
- Treat `deck_context_pack.v1` as the new Agent handoff contract, not a rename of `deck_connector_import.v1`.
- Reuse connector import validation ideas where useful, especially local source pointers and sensitivity checks.
- Add adapter tests only if context pack and connector import need explicit interoperability.

## Allowed files
- `scripts/context_intake/context_pack.py`
- `scripts/deck_master.py`
- `skills/deck-master/schemas/context_pack.schema.json`
- `tests/test_context_pack_import.py`

## Requirements
- Deck Master does not parse PDF/XLSX/PPTX itself.
- Accept Agent-produced JSON.
- Validate schema before import.
- Bad JSON must not overwrite existing `context_manifest.json`.
- Preserve source_id, source_type, origin_type, origin_path, evidence_candidates, sensitivity, publication_status.
- High sensitivity must be visible to Quality Gate / Review Cockpit.
- Import writes typed events.
- Existing legacy events remain readable; this task writes only canonical typed events.

## Tests
- Valid context pack import.
- Duplicate source_id behavior.
- Invalid schema rejection.
- Bad JSON preservation.
- High sensitivity marking.
- Claim graph can consume imported evidence candidates.

## Done
Codex can write `context_pack.json`, Deck Master imports it into a run.

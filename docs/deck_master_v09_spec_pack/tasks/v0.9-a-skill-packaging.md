# Task A — Skill Packaging & Installation

## Goal
Package Deck Master as a Codex-first Agent skill maintained in this repository, installed under `~/.deck-master/current`, and symlinked into external Agent skill directories.

## Scope
Implement:
- `skills/deck-master/`
- `install-skill`
- `validate-skill`
- `uninstall-skill`
- installed source precedence for `~/.deck-master/current/skills/deck-master`

## Allowed files
- `scripts/skills/installer.py`
- `scripts/deck_master.py`
- `skills/deck-master/**`
- `tests/test_skill_installation.py`

## Requirements
- Use symlink by default.
- Real user installs must use `~/.deck-master/current/skills/deck-master` as the default skill source.
- Development repo skill source is only a fallback or explicit `--source-skill-dir` input.
- Agent skill directories contain only the `deck-master` entry symlink.
- Never overwrite a real directory unless explicitly safe.
- `--force` can replace an existing symlink.
- Support `codex`, `claude-code`, `hermes`, `custom`.
- No dependency on any Agent private API.
- CLI outputs JSON.
- Validate `SKILL.md` YAML frontmatter before reporting the skill as installed or valid.
- Typed events or global install log required.

## Tests
- Install into temp skill dir.
- Reinstall idempotent.
- Validate active symlink.
- Validate installed source precedence.
- Reject invalid skill package frontmatter.
- Uninstall only symlink.
- Existing real dir blocks install.
- Unsupported target returns clear error.

## Done
Codex can discover `~/.deck-master/current/skills/deck-master/SKILL.md` through a symlinked skill directory, without coupling the user runtime to a development repository.

# Deck Master Web UI x MainQuest Visual Alignment Spec

日期：2026-06-21  
状态：视觉对齐规范 v1.1  
适用范围：Deck Master localhost Web UI  
设计基线：MainQuest AI Visual Identity System v4.1

## 1. 结论

本轮 localhost Web UI 视觉方向锁定为 MainQuest `Glass Terminal` 深色体系，并映射到`方案项目工作台`场景。目标气质只有一句话：

`高端专业工具，用来推进方案交付判断。`

## 2. 前台气质要求

界面应传达四个感受：

1. 稳定
2. 克制
3. 可判断
4. 可交付

明确回避：

- 演示页感
- 黑客工具感
- 拼装后台感
- 过度营销化装饰

## 3. 视觉基线

### 3.1 背景

- 主底色接近 `#030305`
- 允许 1 到 2 个低强度暖色环境光
- 不允许紫色主导和大面积渐变噪声

### 3.2 玻璃层级

主卡片采用统一玻璃层：

```css
background: rgba(255,255,255,0.03);
backdrop-filter: blur(24px);
border: 1px solid rgba(255,255,255,0.08);
border-top: 1px solid rgba(255,255,255,0.15);
border-radius: 20px;
```

应用位置：

- 项目头部条
- 页面队列栏
- 主舞台
- 处理侧栏
- 底部状态面板

### 3.3 字体

- 标题与正文：`Inter` + 中文系统无衬线
- 数据、时间、ID、计数：等宽字体
- 中文为默认前台语言

## 4. 色彩纪律

建议 token：

```css
--bg-deep: #030305;
--glass-surface: rgba(255,255,255,0.03);
--glass-border: rgba(255,255,255,0.08);
--glass-highlight: rgba(255,255,255,0.15);
--text-primary: #FFFFFF;
--text-secondary: #9CA3AF;
--text-tertiary: #6B7280;
--status-success: #2ECC71;
--status-warning: #F1C40F;
--status-error: #E74C3C;
--accent-action: #FF6B2B;
```

前台语义映射：

- 绿色：已批准、已覆盖、已就绪、可交付
- 黄色：待审阅、待补依据、等待判断
- 红色：阻断、驳回、风险冻结
- 橙色：人工推进动作、下一步

## 5. 组件规则

### 5.1 按钮

- 功能按钮：深色玻璃按钮，`10px` 圆角
- 主动作按钮：白底黑字 CTA，`10px` 圆角
- 禁用态：弱化亮度，不做脏灰对比

### 5.2 状态芯片

状态芯片必须统一四要素：

- 颜色
- 描边
- 文案长度
- 字重

示例：

- `待准备`
- `待审阅`
- `可交付`
- `风险冻结`

### 5.3 数据面板

- 标题小而稳
- 数值清楚
- 不堆无用字段
- 时间、数量、比率全部用等宽字体

### 5.4 主舞台

主舞台是本产品最重要的差异化区域：

- 在前期阶段承载阶段工作区
- 在逐页阶段承载页面预览
- 在交付阶段承载交付预览

周边 chrome 必须克制，不能抢主视觉。

## 6. 版式要求

- 桌面主宽度按 1440 级别优化
- 项目头部条先于侧栏与主舞台
- 左栏更窄，中央最大，右栏稳定
- 留白足够，避免密集控制台观感

## 7. 明确禁止

1. 前台主界面混入内部术语
2. 大面积白卡或浅色模式残留
3. 按钮 pill 化
4. 默认系统字体主导
5. 英文标签压过中文主信息
6. 随意堆砌普通后台卡片

## 8. 视觉验收清单

- [ ] 深色主底稳定
- [ ] 玻璃层级统一
- [ ] 主动作按钮层级清晰
- [ ] 状态芯片颜色纪律一致
- [ ] 数据、时间、ID 使用等宽字体
- [ ] 主舞台始终是首屏视觉重点
- [ ] 左右栏不再有拼装后台感
- [ ] 整体观感达到专业工具水准

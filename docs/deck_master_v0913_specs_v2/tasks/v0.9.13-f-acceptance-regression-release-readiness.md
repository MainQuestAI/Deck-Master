# v0.9.13-F — Acceptance, Regression, Release Readiness

## Goal

Define final acceptance so v0.9.13 proves Deck Master is a single-install Agent-facing Product Capability Suite, not only a set of good-looking Skill directories.

## Final acceptance gates

### Gate 1 — Fresh install full suite

```bash
export HOME=<tmp_home>
deck-master suite-build-release-tree --output "$HOME/.deck-master/current"
deck-master suite-install --target codex --target claude-code
deck-master suite-status --output json
```

Pass only if:

```text
status=ready
full_suite_ready=true
```

### Gate 2 — Setup with suite install

```bash
deck-master setup --workspace <tmp_workspace> --target codex --target claude-code --repair-workspace --install-suite
deck-master setup-status --include-suite --output json
```

Pass only if:

```text
production_ready=true
full_suite_ready=true
workspace_ready=true
```

### Gate 3 — Capability smokes

```bash
deck-master library-status
deck-master quality-gate draft --run-dir <fixture_run>
deck-master generation-session create --run-dir <fixture_run>
deck-master run-generation --run-dir <fixture_run> --dry-run
deck-master render --run-dir <fixture_run> --format html --fixture-safe
```

Each required capability must have fixture-safe smoke and active run writeback.

### Gate 4 — Render / delivery readiness

- Approved queue may exist before render.
- Delivery readiness and benchmark RC readiness require render result when render is required.
- Missing PPT Master readiness blocks full suite readiness.
- Canonical render result path is `render_results/render_result.json`.
- Legacy render result paths may be read as fallback but cannot be the v0.9.13 write target.

### Gate 5 — No private or macOS artifacts

Fail on:

```text
.DS_Store
__MACOSX
.env
*.key
font files
private screenshots
local database files
real customer data
```

### Gate 6 — Codex and Claude Code parity

Both targets must pass:

- fresh install.
- symlink validation.
- migration plan.
- migration apply.
- rollback.
- suite-status.

## Non-acceptance states

The following are not sufficient for final v0.9.13 acceptance:

- `degraded_ready` for required suite.
- metadata-only capability packages.
- missing PPT Master render smoke.
- fake capability smoke that does not write active run state.
- Codex-only install without Claude Code test.
- render result written only to `external_results/render_result.json` or root `render_result.json`.
- Skill docs that tell the user to run raw setup commands without Agent-guided setup ceremony.
- status commands that mutate HOME, workspace, run artifacts, install logs, or file mtimes.

## Validation commands

```bash
python3 -m unittest discover -s tests
git diff --check HEAD
find . \( -name .DS_Store -o -name __MACOSX -o -name .env \) -print
```

All install and smoke commands in CI / QA must run with temporary HOME and temporary agent skill directories. Real local deployment is a separate post-QA step.

## QA report requirements

QA report must include:

- Required suite readiness JSON.
- Codex install matrix.
- Claude Code install matrix.
- Migration and rollback proof.
- PPT Master render smoke result.
- Canonical render result path proof.
- Capability discovery source report.
- Explicit override warning test.
- setup-status / suite-status pure-read proof.
- Agent-driven setup ceremony doc test result.
- No private artifact scan.

## Merge readiness

Merge only when all final acceptance gates pass.

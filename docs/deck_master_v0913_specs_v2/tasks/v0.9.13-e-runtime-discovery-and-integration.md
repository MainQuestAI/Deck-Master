# v0.9.13-E — Runtime Capability Discovery and Integration

## Goal

Route Deck Master runtime calls to bundled required capabilities by default, while preserving explicit CLI override as an intentional advanced path.

## Discovery policy

Default discovery:

```text
1. bundled capability
2. workspace capability override
3. global tool registry
```

Explicit override:

```text
0. explicit CLI override
1. bundled capability
2. workspace override
3. global registry
```

Explicit override must log:

```json
{
  "source": "cli_override",
  "warning": "Explicit capability override used. Output remains subject to Deck Master import contract."
}
```

## Runtime integrations

### Library

```bash
deck-master library-status
deck-master search-library --run-dir <run_dir> --run-id <run_id>
deck-master import-library-selection --run-dir <run_dir> --run-id <run_id> --input <selection.json>
```

Default source: bundled `ppt-library` capability.

### Generation

```bash
deck-master generation-session create --run-dir <run_dir> --run-id <run_id>
deck-master run-generation --run-dir <run_dir> --run-id <run_id>
deck-master generation-session import-results --run-dir <run_dir> --run-id <run_id> --input <result.json>
```

Default source: bundled `ppt-deck-pro-max` capability.

### Render

```bash
deck-master render --run-dir <run_dir> --run-id <run_id> --format html --fixture-safe
deck-master import-render-result --run-dir <run_dir> --run-id <run_id> --input <render_result.json>
```

Default source: bundled `ppt-master` capability.

Render integration must write canonical `render_results/render_result.json`. Runtime may read legacy `external_results/render_result.json` and root `render_result.json` only as fallback.

### Quality

```bash
deck-master quality-gate draft --run-dir <run_dir> --run-id <run_id>
deck-master import-quality-findings --run-dir <run_dir> --run-id <run_id> --input <findings.json>
```

Default source: bundled `ppt-quality-gate` capability for fixture-safe findings and render/delivery audit where applicable.

## Acceptance

- Runtime uses bundled capability when no override supplied.
- CLI override wins when explicitly supplied and records warning.
- Workspace override works and records source.
- Global registry works as fallback.
- No discovery path can bypass run_id/session_id binding or import contract.
- Run-state can see capability output and block/allow next stage.
- run-state resolver, benchmark runner, and Review Cockpit readiness read canonical render result before legacy fallback.

## Tests

- Bundled-first default for all four capabilities.
- Explicit override wins and logs warning.
- Workspace override selected when no CLI override.
- Global registry fallback.
- Broken bundled capability blocks full suite readiness.
- Render missing blocks delivery / RC readiness.
- Canonical render result enables delivery / RC readiness when other gates pass.

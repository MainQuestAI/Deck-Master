# v0.9.13-A — Product Capability Manifest, Release Tree, Full Suite Readiness

## Goal

Create the product capability manifest and release tree builder, and define `full_suite_ready=true` as the final required-suite readiness target.

Task A defines readiness semantics. It does not need to make `full_suite_ready=true` pass before Task B creates executable capability contracts.

## Scope

- `product-capability-manifest.json` schema and default manifest.
- `suite-build-release-tree` command.
- Required capability list.
- Full suite readiness calculation.
- `.DS_Store` / `__MACOSX` exclusion.
- Pure-read `suite-status` inspection behavior.

## Required capabilities

```text
deck-master
deck-planner
deck-review
ppt-master
ppt-library
ppt-deck-pro-max
ppt-quality-gate
```

`deck-learning` remains optional in v0.9.13.

## Public CLI

```bash
deck-master suite-build-release-tree --output ~/.deck-master/current
deck-master suite-status --output json
```

## Manifest minimum fields

```json
{
  "schema_version": "deck_master_product_capability_manifest.v1",
  "runtime_shape": "agent_facing_local_first",
  "required_capabilities": [],
  "optional_capabilities": [],
  "core_skills": [],
  "product_capability_skills": [],
  "capability_policy": {
    "required_suite_must_be_full_ready": true,
    "outputs_must_write_back_to_run": true,
    "external_override_allowed": true
  }
}
```

## Acceptance

- Fresh temp HOME can build release tree.
- Release tree contains required skills and capability directories.
- `suite-status` returns required capability readiness separately from optional capability readiness.
- Before executable capability contracts exist, required capability missing / metadata-only packages must produce `full_suite_ready=false`.
- Final fixture acceptance requires `full_suite_ready=true` only after Task B and Task F gates are available.
- `degraded_ready` is not accepted for final v0.9.13 readiness.
- Generated release tree contains no `.DS_Store`, `__MACOSX`, `.env`, font files, screenshots, private artifacts.
- `suite-status --output json` must not write HOME files, install logs, workspace files, run artifacts, or change mtimes.

## Tests

- Manifest schema validation.
- Release tree idempotency.
- No private/macOS artifacts.
- Required capability missing -> `full_suite_ready=false`.
- Optional capability missing -> may remain degraded only for optional status, not required status.
- `suite-status` pure-read mtime check under temporary HOME.

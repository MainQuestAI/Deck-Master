# v0.9.13-D — Suite Install, Legacy Migration, Setup

## Goal

Make Deck Master setup install and validate the full required suite for Codex and Claude Code, with safe migration and rollback for legacy real directories.

## Public CLI

```bash
deck-master suite-install --target codex --target claude-code
deck-master suite-repair --target codex --target claude-code
deck-master suite-migrate-legacy-skills --target codex --target claude-code --plan
deck-master suite-migrate-legacy-skills --target codex --target claude-code --apply --plan-file <plan.json>
deck-master suite-migrate-legacy-skills --target codex --target claude-code --rollback --rollback-id <id>
deck-master setup --workspace <path> --target codex --target claude-code --repair-workspace --install-suite
```

## Migration rules

- Never overwrite a real directory without migration plan.
- Identify known legacy skills.
- Back up migrated real directories.
- Replace only after explicit apply.
- Rollback restores original real directories.
- Migration logs are JSON and auditable.

## Migration plan minimum schema

```json
{
  "schema_version": "deck_master_legacy_skill_migration_plan.v1",
  "target": "codex|claude-code",
  "agent_skill_dir": "...",
  "created_at": "...",
  "actions": [
    {
      "skill": "ppt-library",
      "current_path": "...",
      "current_type": "real_directory|foreign_symlink|broken_symlink|missing",
      "recognized_as": "legacy_ppt_library_skill|unknown_real_directory",
      "target_link": ".../.deck-master/current/skills/ppt-library",
      "backup_path": ".../.deck-master/current/migration/backups/<id>/ppt-library",
      "action": "backup_and_replace_with_symlink|repair_symlink|no_op|manual_action",
      "safe_to_apply": true,
      "warnings": []
    }
  ],
  "rollback": {
    "rollback_id": "...",
    "command": "deck-master suite-migrate-legacy-skills --target <target> --rollback --rollback-id <id>"
  }
}
```

## Acceptance

- Fresh install works for Codex and Claude Code.
- Migration plan works for Codex and Claude Code.
- Apply creates symlinks to release tree.
- Rollback restores previous real directories.
- `setup --install-suite` builds release tree, links skills, validates suite, validates workspace.
- Required suite readiness after fresh install is `full_suite_ready=true`.
- `degraded_ready` is accepted only for optional capabilities or intentionally broken test fixtures.
- CI / QA install tests use temporary HOME and temporary agent skill directories.
- Real user install is verified only in a separate deployment step.
- `setup-status --include-suite` and `suite-status` remain pure-read; mutating behavior belongs to setup / suite-install / suite-repair / suite-migrate only.

## Tests

- Codex fresh install.
- Claude Code fresh install.
- Codex migration plan/apply/rollback.
- Claude Code migration plan/apply/rollback.
- Real directory not overwritten without apply.
- Setup with `--install-suite` returns production-ready setup with full suite ready.
- Migration plan schema validation.
- Temporary HOME install does not touch real `~/.deck-master`.

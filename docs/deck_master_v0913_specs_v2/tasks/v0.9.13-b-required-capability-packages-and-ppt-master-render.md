# v0.9.13-B — Required Capability Packages and PPT Master P0 Render Path

## Goal

Create required Product Capability packages with minimum executable contracts. PPT Master must be a real P0 render capability, not a placeholder.

## Scope

Required package shape:

```text
product_capabilities/<capability>/
  capability.yaml
  SKILL.md
  contracts/
  adapters/
  smoke/
```

Capabilities:

```text
ppt-master
ppt-library
ppt-deck-pro-max
ppt-quality-gate
```

## Required executable contract

Every required capability must support:

```text
status
doctor
schema
smoke --fixture
writeback
```

Implementation may be a wrapper, Python module, vendored source, or generated fixture-safe runner. Metadata-only package is not ready.

## PPT Master P0 render path

### Required CLI

```bash
deck-master render-status --run-dir <run_dir>
deck-master render --run-dir <run_dir> --format html --fixture-safe
deck-master import-render-result --run-dir <run_dir> --input <render_result.json>
```

### Required artifacts

```text
render_session.json
render_results/render_result.json     # canonical v0.9.13+ path
rendered/index.html
```

### Minimum behavior

- Reads active run `preview_manifest.json`.
- Builds a fixture-safe HTML render artifact from run-local previews.
- Writes `deck_render_result.v1`.
- Imports render result into `render_results/render_result.json`.
- Reads legacy `external_results/render_result.json` and root `render_result.json` only as compatibility fallback.
- Normalizes any legacy render result import into `render_results/render_result.json`.
- Updates preview / run-state / benchmark readiness to read canonical first, then legacy fallback.
- Run-state blocks delivery / benchmark RC if render is required but missing.

## Capability-specific minimums

### PPT Library

- Fixture-safe selection smoke.
- Canonical `deck_master_ppt_library_selection.v1` output.
- Import writes canonical and legacy compatibility artifacts.

### PPT Deck Pro Max

- Fixture-safe generation smoke.
- Generation session result must include `run_id + session_id`.
- Direct result import compatibility remains limited to legacy path.

### PPT Quality Gate

- Fixture-safe findings smoke.
- Canonical `deck_master_quality_findings.v1` input.
- Import converts to Deck Master quality report and recomputes blocking state.

## Acceptance

- All required capabilities have status/doctor/smoke/schema/writeback.
- PPT Master smoke produces real run-local artifact and render result.
- Required capability missing or metadata-only -> full suite not ready.
- All writeback paths go through Deck Master import/state contract.
- Render result schema is `deck_render_result.v1`; legacy `deck_master_render_result.v1` is treated as alias only if explicitly normalized.
- No new code writes canonical render result to `external_results/render_result.json` or root `render_result.json`.

## Tests

- PPT Master render smoke creates `rendered/index.html` and canonical `render_results/render_result.json`.
- Legacy `external_results/render_result.json` import is normalized into canonical render result.
- run-state resolver and benchmark runner recognize canonical render result.
- Missing PPT Master executable contract -> suite not full ready.
- PPT Library / Deck Pro Max / Quality Gate fixture smokes pass.
- Render missing blocks delivery readiness.

# Deck Master v0.9.14 Real Production Closure

Date: 2026-06-22

## Summary

This release closes the current production-readiness loop for Deck Master runtime contracts, generation handoff, artifact truth, final readiness, release packaging, benchmark metadata, and RC gate reporting.

## Added

- `deck_generation_result.v2` import and validation.
- Production Agent dispatch package with `awaiting_agent_execution`.
- PPT-Deck-Pro-Max Deck Master bridge commands in the companion repository.
- Build/runtime artifact output with manifest, checksum, media type, byte size, and editability metadata.
- Shared artifact validator for HTML, PDF, PNG, JPEG, SVG, PPTX, JSON report artifacts, path safety, checksum, stub content, and stale source checks.
- Delivery validation lineage via `final_version_lineage.json`.
- Single final readiness gate via `delivery/final_readiness.json`.
- Client export enforcement based on final readiness.
- Fixture/dev/production isolation for production and benchmark paths.
- Self-contained release tree with `release-manifest.json`, `deck_capability_lock.json`, and `SHA256SUMS`.
- Staged release install, verification, activation, rollback, and release smoke.
- Three sanitized real benchmark metadata cases.
- Benchmark aggregate report and contract.
- RC gate report and CI RC gate step.

## New Commands

```bash
python3 scripts/deck_master.py release-build --output <release_dir> --force
python3 scripts/deck_master.py release-smoke --release-root <release_dir>
python3 scripts/deck_master.py release-rollback
python3 scripts/deck_master.py final-readiness --run-dir <run_dir>
python3 scripts/deck_master.py benchmark-aggregate-report --benchmark-dir benchmarks --force
python3 scripts/deck_master.py rc-gate --output-dir <out_dir> --benchmark-dir benchmarks --skip-browser-smoke --force
```

Existing compatibility commands remain available, including `suite-build-release-tree`.

## Known Boundaries

- Browser smoke is optional by default. Use `--require-browser-smoke` when Playwright/browser runtime is available and required.
- Private benchmark source documents are excluded from the repository. Only sanitized metadata is committed.
- Current build/runtime output closes contracts and lineage first. Native visual-fidelity rendering remains a later adapter task.

## 二、真正的 1.0.0 缺口：可复现 / 可追溯 / 可发布

功能已闭合，但**外部用户 clone 后无法复现这些绿色证据**——这才是 1.0.0 的真实缺口：

### 2.1 证据不可复现（最深缺口）

| 证据 | 本机状态 | clone 后 | 根因 |
| --- | --- | --- | --- |
| `rc_reports/rc_gate_report.json` | 存在，pass | **不存在** | 根 `.gitignore` 第 26 行排除 `rc_reports/` |
| `benchmarks/results/`（含 aggregate + 3 real 报告） | 存在，`report_ready` | **不存在** | `benchmarks/.gitignore` 排除 `results/`、`benchmark_runs/`、`private_sources/` |
| 3 个 real case 源材料 | 本机 `~/deck-master-local-benchmarks/…` | **不存在** | `raw_source_policy=local_path_only`，context_pack 指向本机路径，不提交 |
| real case 元数据 | — | **已提交**（`benchmarks/cases/real_*/benchmark_case.json`，`case_type=real_metadata`） | 仅元数据公开，无可运行源材料 |

**关键推论（两份复核都未点透）**：在 CI 里跑 `rc-gate` 时，`benchmark_aggregate` 检查会**失败**——fresh clone 的 `benchmarks/results/` 为空，real case 又无源材料可重跑，aggregate 只能是 `blocked`（0 成对报告），不可能是 `report_ready`。唯一能完整跑的是 `retail_fixture`（`library_mode=fixture`，不计入 `min_real_cases=3`）。

> 因此复核 B 的「Sprint 1：M2 门禁进 CI，跑 rc-gate + benchmark aggregate」存在**结构性阻断**：CI 无法 pass benchmark_aggregate，除非先解决可复现证据分层。

### 2.2 PPT Master 绑定可复现性

- 公开仓库存在：`github.com/hugohe3/ppt-master`
- 但当前绑定指向**本机 worktree** `/Users/.../ppt-master-deck-master-backend`，分支 `codex/deck-master-backend-certification`（feature 分支，非 main/tag）
- 外部用户无法通过「机器路径」复现 bind/verify；需文档化「clean-env bind 路径」并 pin 到 tag/main + sha

### 2.3 CI 未强制 M2/RC 门禁

- 当前 CI（`.github/workflows/ci.yml`）只跑 unittest + autoplan smoke + fixture smoke；**不跑** rc-gate / benchmark aggregate / release install-rollback smoke
- 本机绿 ≠ PR 上绿

### 2.4 开源治理 + 产品收口 + 工程结构

| 项 | 状态 |
| --- | --- |
| `docs/known-limitations.md` | 仍 `preview.1 / 0.9.14a1`，未同步 preview.2 |
| GitHub 社区文件 | 6 项全缺（ISSUE_TEMPLATE / PR template / CODEOWNERS / dependabot / ROADMAP） |
| Review Desk 产品级 | M1 preview；README 无截图/GIF；无 `docs/user-guide.md`；`DESIGN.md` 系统已定义但 UI 未全量对齐 |
| Makefile / Dockerfile / .devcontainer | 均无（Dockerfile 直接关系到可复现性，应提升优先级） |
| `scripts/deck_master.py` | 3609 行，长期维护风险 |
| 版本一致性 | `suite-status` 报 `suite_version=1.1.0`，而 pyproject 为 `0.9.14a2`；`docs/releases/` 有 `v1.1.0-release-notes.md` 却无 `v1.0.0.md`——需澄清 |

## Key Interactions to Verify
- index ‚Üí search round-trip: index a known .pptx, search for text from it, confirm it's found in results
- Incremental index: index directory, touch one file, re-index ‚Üí only changed file processed
- Embedding API failure: search while API returns 500 ‚Üí graceful error message, not crash

## Edge Cases
- Empty slide (image-only): text_content="" but slide still indexed with screenshot
- Password-protected PPTX: skipped with warning, doesn't block batch
- 10,000-character slide text: embedding API handles long input without truncation loss
- Concurrent index + search: SQLite handles WAL mode, reader doesn't block writer

## Critical Paths
- Single PPTX ‚Üí index ‚Üí search for its content ‚Üí confirmed in top-3 results
- Batch directory ‚Üí all .pptx indexed ‚Üí status shows correct counts
- Search with empty library ‚Üí graceful empty result, no crash

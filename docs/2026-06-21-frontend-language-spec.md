# 方案项目工作台前台语言 Spec

日期：2026-06-21  
状态：v0.2  
适用范围：Deck Master localhost Web UI

## 1. 目标

前台语言要让非技术用户看见的是方案交付流程，看不见内部运行结构。

## 2. 锁定词表

### 2.1 前台保留词

- 方案项目
- 页面处理
- 来源与依据
- 风险与缺口
- 审批记录
- 交付预览
- 确认交付

### 2.2 禁用词

- run
- audit
- workspace
- review cockpit
- export queue
- runtime readiness
- 原始状态枚举

## 3. 阶段文案

| 阶段 | 前台定义 | 下一步口径 |
| --- | --- | --- |
| 待准备 | 项目已创建，但还没有进入可审状态。 | 先形成首版页面与预览。 |
| 生成中 | 页面内容和预览仍在形成。 | 等待生成完成，再进入逐页处理。 |
| 待审阅 | 页面已可见，但关键判断还未完成。 | 先处理阻断页。 |
| 待补依据 | 内容结构已成形，但依据不足。 | 补齐依据后再做判断。 |
| 待审批 | 页面处理已基本完成，等待明确拍板。 | 完成审批记录。 |
| 可交付 | 当前页面、门禁与交付预览已满足交付条件。 | 查看交付预览并确认交付。 |
| 已交付 | 当前项目已记录交付结果，可继续回看。 | 回看交付成片与处理记录。 |
| 风险冻结 | 当前存在高优先级风险，流程暂不继续。 | 先解除阻断。 |

## 4. 动作文案

- `批准页面`
- `驳回页面`
- `请求补依据`
- `升级审批`
- `记录备注`
- `提交审批`
- `确认交付`

禁止把动作写成技术动作名或命令短语。

## 5. 空态文案原则

空态统一采用两句结构：

1. 告诉用户当前没有什么
2. 告诉用户下一步做什么

示例：

- `当前还没有页面。先补齐页面生成、预览构建或前置资料，再回到工作台处理。`
- `当前还没有交付级预览。完成交付渲染后，这里会展示最终交付预览。`

## 6. 错误解释原则

- 先说影响
- 再说建议动作
- 不直抛底层文件名给主标题

允许在说明区保留文件名或路径，但必须是补充信息。

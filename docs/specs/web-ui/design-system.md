# Deck Master Review Desk Design System

日期：2026-06-23  
适用范围：`scripts/preview/static/`

## 1. 产品定位

- 这是 Deck Master 的本地 Review Desk，服务一次 Deck run 的审查、返修和交付判断。
- 首屏服务对象是解决方案架构师和专业顾问，核心任务是快速判断“当前卡在哪一页、为什么卡住、下一步该怎么推进”。
- 默认使用桌面端，重点照顾 13-16 寸笔记本。

## 2. 视觉方向

- 记忆点固定为“严肃工具感”。
- 视觉风格固定为 `Industrial / Utilitarian x Brutally Minimal`。
- 层次依靠排版、发丝边框、信息密度和状态色建立，不依赖玻璃质感、漂浮光球或大面积装饰。
- 暖色只作为信号灯使用，不能扩散成氛围色。

## 3. 字体体系

- 标题：`Satoshi`，强调几何感和统领感。
- 正文与界面：`Geist`，承担主要阅读和控件表达。
- 数据与短枚举：`IBM Plex Mono`，承担页码、状态、run id、指标等结构身份标记。
- 主字体禁用：`Inter`、`Roboto`、`Arial` 一类默认收敛字体。

## 4. 颜色体系

- 深色是默认主路径。
- 主强调色：`#E09043`，仅用于主动作、当前态、阻断提醒。
- 中性底色：
  - `--ink-base: #0B0E12`
  - `--surface-1: #11151B`
  - `--surface-2: #161B22`
  - `--surface-3: #1B212A`
  - `--hairline: #232A33`
- 文本：
  - `--text-primary: #E6E9EE`
  - `--text-secondary: #8A93A0`
  - `--text-tertiary: #5A626C`
- 语义色：
  - success `#6FAE6F`
  - warning `#D9A441`
  - danger `#C75450`
  - info `#5B8DB8`

## 5. 布局原则

- 首屏围绕“当前页”展开，run 级信息降到顶部状态带和底部抽屉。
- 固定三栏：
  - 左栏：任务目录、筛选、页面队列
  - 中栏：当前页预览、主动作条、交付预览入口
  - 右栏：职责、来源、证据、风险、审批
- 底部抽屉承接 run 级就绪度、论点覆盖、事件日志、Build Skill、Artifact、Export。
- 圆角保持克制，避免气泡化界面。

## 6. 交互原则

- 始终维持一个主动作，其他动作降级到次动作。
- 当前阶段如果还在准备或生成，页面级动作必须收起或禁用，且要说清楚原因。
- P0 / P1 阻断必须在三个地方联动表达：
  - 顶部阻断标记
  - 中栏主动作条
  - 右栏风险块
- 语言切换按钮保留，但在真实双语未完成前必须是禁用态。
- planned run、preview 未就绪、delivery preview 缺失时，只能给安全空态，不能白屏、不能泄露本机路径。

## 7. 用户用语

- 用户侧统一说 `Build / Deck Builder`。
- 诊断信息里可以保留 `PPT Master backend` 这样的底层名称。
- 阻断文案优先讲业务语义，例如“最终文件仍包含内部制作语言，需要返修”。

## 8. 不做的事

- 不把临时设计资产、截图基线、实验性双语实现带进主线。
- 不为视觉统一去改动运行时契约语义。
- 不让 UI 文案回退到内部工程术语。

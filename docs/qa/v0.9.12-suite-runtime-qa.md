# Deck Master v0.9.12 Suite Runtime QA

## Scope

本轮 QA 覆盖 Stack A、Stack B、Stack C 三段连续开发：

- Stack A：suite runtime foundation、纯读 setup/suite status、suite install/repair。
- Stack B：PPT Library、PPT Quality Gate、PPT Deck Pro Max handback adapter 与 import log。
- Stack C：library feedback event queue、Review Cockpit 可见状态、run metrics 与 release closure。

## Acceptance Checks

- `setup-status --include-suite --output json` 和 `suite-status --output json` 在临时 HOME 下不写入 `.deck-master`。
- setup 未 ready 时，production run 创建继续阻断。
- malformed、缺失、真实目录冲突、CLI 缺失的 companion skill 能返回明确状态和下一步。
- library selection 导入同时写入 `external/ppt_library/library_results.json` 与 legacy `library_results/selection.json`。
- quality findings 导入生成 `deck_quality_report.v1`，`blocking` 默认映射为 P1 并阻断 delivery。
- Deck Pro Max handback 导入校验 `run_id` 和 `session_id`，成功 refresh preview 后进入 `quality_required`。
- rejected input 不覆盖旧 artifact，并写入 rejected import log。
- feedback 默认写 run-local event queue，`--apply` 保持 experimental。
- Review Cockpit external results 面板能看到 suite readiness、import log、blocking quality、pending feedback。
- metrics 汇总 import log、pending feedback、blocking quality 状态。

```bash
python3 -m unittest tests.test_skill_installation tests.test_setup_enforcement
python3 -m unittest tests.test_ppt_library_client tests.test_sourcing_decider tests.test_external_quality_review tests.test_generation_session_bridge tests.test_generation_handback
python3 -m unittest tests.test_library_feedback_queue tests.test_run_metrics tests.test_preview_server tests.test_review_workbench

git diff --check
```

## Degraded Conditions

- `ppt-master` 仍作为 optional companion；缺失时只标记 `render_export=optional_missing`。
- `record-library-feedback --apply` 暂不写外部库，只保留 run-local event queue。
- Review Cockpit 本轮只做最小状态区，完整 UI 重设计留到后续版本。

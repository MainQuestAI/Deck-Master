# Deck Master v0.9.13 Product Capability Suite QA

## Scope

v0.9.13 验证 Deck Master 是否已经从“单一主 skill + 外部能力依赖”收敛为可一次安装的 Product Capability Suite。

## Acceptance Gates

- `suite-build-release-tree` 必须产出 `skills/`、`capabilities/`、`contracts/`、`reference-packs/`、`bin/deck-master` 和 `product-capability-manifest.json`。
- `suite-install` 必须安装 required suite skills：`deck-master`、`deck-planner`、`deck-review`、`ppt-master`、`ppt-library`、`ppt-deck-pro-max`、`ppt-quality-gate`。
- `suite-status` 的 final acceptance 必须是 `ready` 和 `full_suite_ready=true`。
- setup ceremony 必须由 Agent 驱动，并支持 `setup --install-suite`。
- PPT Master render 必须写 canonical `render_results/render_result.json`。
- legacy `external_results/render_result.json` 和 root `render_result.json` 只能作为读取兜底。
- `generation-session` 默认发现 bundled `ppt-deck-pro-max` capability；显式 CLI override 必须带 warning。
- `benchmark-rc-report` 在 case 声明 `requires_render_result=true` 且缺 render result 时必须阻断。

## Verification Commands

```bash
python3 -m unittest tests.test_skill_installation
python3 -m unittest tests.test_render_runtime tests.test_orchestration_enforcement tests.test_uat_render_tool
python3 -m unittest tests.test_generation_session_bridge
python3 -m unittest tests.test_benchmark_runner tests.test_benchmark_report
python3 -m unittest discover -s tests
git diff --check HEAD
```

## Evidence To Capture Before Release

- Temporary HOME `setup --install-suite --target codex --target claude-code` result.
- `suite-status --output json` with `full_suite_ready=true`.
- Render smoke output path proving `render_results/render_result.json`.
- RC benchmark blocked-before-render and passed-after-render proof.
- QA skill review result.

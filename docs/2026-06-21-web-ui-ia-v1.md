前置依据：

## 1. 方向锁定（结论）

首屏主视角：**围绕当前页展开。**

- run 级信息从“首屏主轴”降为“顶部一条状态带 + 可展开抽屉”。
- 首屏黄金位让给当前页审查：预览 + 判断 + 动作。
- Web UI 身份收敛为“审稿桌”，不退回“项目工作台”。

这一条是后续所有版式、动作、视觉决策的前提。如未推翻，下轮实现以此为准。

## 2. 信息层级

沿用 audit 的 A/B/C 三层，落到具体语义：

### A 层：当前任务（首屏 5 秒内必须看懂）
- 当前 run 名称 + run id
- 当前阶段
- 当前最需要处理的页面（阻断页优先，否则上次查看页）
- 当前是否存在阻断（P0/P1）
- 当前建议动作

### B 层：页面判断（做审查决定时必须看）
- 预览图
- 页面职责
- 核心论点
- 来源决策 + 证据充足度
- 风险等级
- 审批动作（一个主动作 + 若干次动作）

### C 层：run 全局补充（有价值但不抢首屏）
- 论点覆盖（claim-coverage）
- run 就绪度（run-readiness）
- 事件日志 / 活动流（activity-list）
- 交付级预览
- 审批记录列表
- Build Skill 步骤 / Artifact handback / Export queue

C 层统一收进底部抽屉或顶部展开区，不持续占用首屏。

## 3. 新首屏布局（低保真）

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ 顶部状态带（一条，不再是四张同权卡片）                                         │
│ Workspace · Run 标题 + id · 阶段 · 下一步 · 阻断标记 · Draft Gate · 导出 · 语言 │
│  —— 点击阻断标记或阶段 → 展开顶部 run 状态抽屉（C 层入口）                      │
├──────────┬────────────────────────────────────────────────┬─────────────────┤
│ 左栏 窄    │ 中栏 大（视觉中心）                              │ 右栏 决策台       │
│          │                                                │                 │
│ 页面筛选   │ 页码 / 章节 / 页面标题  +  主动作条              │ 页面职责          │
│ 页面大纲   │ ┌────────────────────────────────────────────┐ │ 核心论点          │
│ 状态计数   │ │                                            │ │ 来源 · 证据       │
│          │ │            当前页预览图                       │ │ 风险等级          │
│ 专注找页   │ │            （更大、更安静）                  │ │ ─────────────    │
│ 与筛选     │ │                                            │ │ 审批（主动作）     │
│          │ └────────────────────────────────────────────┘ │ 次动作区          │
│          │ 上一页 / 下一页 / 全屏                            │ 备注              │
├──────────┴────────────────────────────────────────────────┴─────────────────┤
│ 底部抽屉（默认收起，C 层全部归此）                                              │
│ Tabs：就绪度 · 论点覆盖 · 事件日志 · Build Skill · Artifact · Export queue       │
└─────────────────────────────────────────────────────────────────────────────┘
```

与现状的关键差异：
- 顶部从“四张同权 header-card”压成“一条状态带”，阻断标记单独成点。
- 中栏底部不再常驻三块 bottom-panel，收进底部抽屉。
- 右栏决策动作收敛为“一个主动作 + 若干次动作”，不再平铺多个同权按钮。
- 预览图外围容器收窄，让预览本身成为唯一视觉张力点。

## 5. 三栏权重重排

- 左栏更窄：专注“找页 + 筛选 + 状态计数”，移除 run 切换等低频项到顶部或菜单。
- 中栏更大：预览图占主导，预览上方加一条“主动作条”，把当前页建议动作压到预览同视线。
- 右栏改为决策台：围绕“能不能过 + 为什么”组织，动作收敛为主动作 + 次动作。

## 6. 动作收敛（主动作 + 次动作）

映射到现有 `.action-grid` 与 `#approval-content`：

| 现有动作 | 归属 | 说明 |
|---|---|---|
| 批准页面 / 驳回页面 | 主动作 | 当前页审查主路径；P0/P1 时主动作条直接反映阻断后果 |
| 请求补证据 | 主动作（条件） | 页面证据不足时作为候选主动作出现 |
| 替换来源 / 转生成页 / 锁定历史页 | 次动作 | 默认收在次动作区，视觉权重低于主动作 |
| 升级审批 | 次动作 | 页面初审完成后再显著出现，不在首屏争抢注意力 |
| 确认交付 | 顶部唯一高权重按钮 | 仅在真正到达 export_ready 阶段时升为顶部主动作 |
| 新建项目 | 顶部次入口 | 降权，不与当前 run 审查争抢注意力 |

补充状态（audit 4.5，本轮结构层先占位，视觉/交互实现后续落地）：
- skeleton loading 替代文字等待
- 强阻断态：P0/P1 直接控制主动作条状态与说明
- 完成态：某页通过后给出明确“下一页建议”，而非仅更新状态

## 7. 落地映射（旧 → 新容器）

为下轮实现提供可直接对照的节点映射：

- `.workspace-header` → 拆为 `.status-bar`（一条带）+ 可展开 `.status-drawer`
- `.header-grid` → 删除，内容并入 `.status-bar`
- `.workspace-grid` → 保留三栏，重排权重（左窄 / 中大 / 右决策台）
- `.center-column .workspace-bottom-grid` → 删除常驻，移入底部 `.bottom-drawer`
- `.preview-panel` → 保留并扩大，预览顶加 `.action-bar`（主动作条）
- `.decision-rail` → 保留为决策台，内部 `.decision-block` 重排为 职责 → 来源·证据 → 风险 → 审批(主动作) → 次动作 → 备注
- 新增 `.bottom-drawer`（含 Tabs：就绪度 / 论点覆盖 / 事件日志 / Build Skill / Artifact / Export）

## 8. 本轮结构层不做的事

- 不改视觉 token 与字体（留到视觉语言重做阶段）。
- 不引入新弹窗或浮层。
- 不动 `.ambient-light-container` 等装饰层。
- 不改双语语言包结构。
- 不做大面积动效。

## 9. 验收点

结构层落地后，以下可验证：

- 首屏不再出现 run 级三块常驻面板（就绪度/论点覆盖/活动流）。
- 顶部为一条状态带，阻断标记单独成点，可展开查看 C 层。
- 右栏动作区只有一个主动作，其余为次动作。
- P0/P1 风险能联动顶部阻断标记、中栏主动作条、右栏风险块三处。
- 底部抽屉默认收起，失败或阻断时可自动提示展开。
- 预览图成为中栏唯一视觉张力点，外围容器收窄。
- 现有验收标准不回退：5 秒判断 run 是否可继续、30 秒找到所有阻断页、首次打开进入第一页、重新打开恢复上次页。

# Playbook — External Quality Review

How to execute an external quality review task generated by Deck Master.

## Prepare Review Task

```bash
python3 scripts/deck_master.py prepare-quality-review \
  --run-id <run_id> \
  --scope semantic,visual,client-readiness
```

Task files appear at `quality_review_tasks/<scope>_review_task.json`.

## Task Inputs

Each task references:

- `deck_brief.json`
- `claim_evidence_graph.json`
- `page_tasks.json`
- `preview_manifest.json`
- `quality_reports/`

Read these to understand the current run state before reviewing.

## Review Dimensions

- `claim_evidence_alignment` — does each page's evidence support its claim?
- `consulting_style_expression` — is the tone appropriate for solution consulting?
- `client_readability` — can the client decision-maker follow the argument?
- `page_job_clarity` — does each page have a clear job in the narrative arc?
- `decision_readiness` — does the deck drive toward a client decision?

## Output Schema

Write result per `schemas/external_quality_review_result.schema.json`:

```json
{
  "schema_version": "deck_external_quality_review.v1",
  "run_id": "<run_id>",
  "reviewer": "codex",
  "scope": "semantic",
  "summary": {
    "status": "rework_required|conditional_pass|pass",
    "blocks_delivery": true,
    "p0_count": 0,
    "p1_count": 0,
    "p2_count": 0
  },
  "findings": [
    {
      "finding_id": "ext_semantic_001",
      "severity": "P1|P2",
      "page_id": "<beat_id>",
      "dimension": "<dimension>",
      "message": "...",
      "repair_instruction": "...",
      "refs": ["..."]
    }
  ]
}
```

## Import Result

```bash
python3 scripts/deck_master.py import-quality-review \
  --run-id <run_id> --input external_quality_review.json
```

P0/P1 findings will block client export until repaired or overridden.
